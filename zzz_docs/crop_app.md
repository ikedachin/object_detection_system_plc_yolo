# crop_appアプリケーション プログラム設計書

## 1. システム概要

本アプリは、物体検出用データセット作成のための画像切り抜き（クロップ）処理を行うDjangoアプリケーションです。  
プロジェクト単位で切り抜きセッション・画像・領域・進捗を管理し、バッチ切り抜きやテンプレート機能も提供します。

---

## 2. 全体フロー図

```
[プロジェクト選択]
      ↓
[切り抜きセッション作成]
      ↓
```

## 3. 主要関数・API詳細

### 3.1 セッション・画像管理

- **crop_tool(request)**  
  切り抜きツールのメイン画面表示。アクティブプロジェクト・セッション一覧を取得。
- **create_crop_session(request)**  
  新しい切り抜きセッションを作成。プロジェクト・セッション名等を登録。
- **get_session_images(request)**  
  セッション内の画像一覧を取得。
- **add_crop_image(request)**  
  セッションに画像を追加。画像情報をDBに登録。
- **save_crop_area(request)**  
  切り抜き領域を保存し、切り抜き画像を生成・保存。

### 3.2 プロジェクト・テンプレート管理

- **get_project_sessions(request)**  
  プロジェクトの切り抜きセッション一覧を取得。
- **get_crop_templates(request)**  
  プロジェクトの切り抜きテンプレート一覧を取得。

### 3.3 バッチ・補助API

- **crop_and_save_all(request)**  
  指定bboxでプロジェクト内全画像を一括切り抜き・保存。
- **process_crop(request), process_batch_crop(request)**  
  画像単体・バッチでの切り抜き処理。
- **save_bounding_box_yaml(request)**  
  切り抜き領域情報をYAML形式で保存。

## 4. 各関数の詳細説明

### crop_tool(request)
- **目的**: 切り抜きツール画面の表示
- **処理**:
  1. アクティブプロジェクト・全プロジェクト取得
  2. セッション一覧取得
  3. テンプレートへデータ渡し
- **返却**: HTML

### create_crop_session(request)
- **目的**: 新規切り抜きセッション作成
- **処理**:
  1. POSTデータ受信
  2. セッションDB登録
  3. 統計情報更新
- **返却**: JSON

### get_session_images(request)
- **目的**: セッション内画像一覧取得
- **処理**:
  1. セッションIDから画像フォルダ探索
  2. 画像ファイル一覧返却
- **返却**: JSON

### add_crop_image(request)
- **目的**: セッションに画像追加
- **処理**:
  1. 画像存在確認・寸法取得
  2. CropImage DB登録
- **返却**: JSON

### save_crop_area(request)
- **目的**: 切り抜き領域保存・画像生成
- **処理**:
  1. 領域情報DB登録
  2. 切り抜き画像生成・保存
  3. 進捗・統計更新
- **返却**: JSON

### crop_and_save_all(request)
- **目的**: 全画像一括切り抜き
- **処理**:
  1. POSTデータ受信
  2. フォルダ内画像を指定bboxで一括切り抜き
  3. croppedフォルダに保存
- **返却**: JSON

### get_project_sessions(request), get_crop_templates(request)
- **目的**: プロジェクトのセッション・テンプレート一覧取得
- **処理**: DBから該当情報を取得し返却
- **返却**: JSON
